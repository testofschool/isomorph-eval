\section{Experimental Setup}
\label{sec:setup}

\subsection{Evaluation Dataset}
We apply the Isomorphic Engine (Section~\ref{sec:engine}) to the
GSM8K test set~\cite{cobbe2021gsm8k}, parsing all 1{,}319 problems
via the rule-based annotation parser. After applying the strict
validation pipeline---filtering problems with fractional leaf values,
value-conflation between inputs and implicit constants, non-integer
variant answers, negative intermediates, excessively large answers
($>10^8$), and text-replacement mismatches---the final dataset contains
\textbf{667 original items with 3{,}199 verified $\tau$-isomorphic
variants} (3{,}866 evaluation entries total, $\sim$4.8 variants per
item). We evaluate a \textbf{stratified subset of 100 items} with
480 variants (580 evaluation entries per model), selected by uniform
sampling across answer-magnitude quantiles as a proxy for difficulty.
Each item is limited to 5~variants.

\paragraph{Independent answer verification.}
Initial evaluation on all 100~items revealed a uniform accuracy gap
across all models, including an intended control model (GPT-OSS 120B,
which was not trained on GSM8K). Investigation identified
annotation-chain replay bugs in 40 of 100~items: the mechanism that
computes variant answers by substituting values into the original
solution chain fails when the same numeric value serves multiple
semantic roles (value conflation) or when substitution mappings are
applied inconsistently. To address this, we developed an independent
verification engine using multi-model consensus: for each variant, if
$\geq 2$ of 5~evaluated models agree on an answer that differs from
the stated expected answer, the variant is flagged as broken. This
procedure identified 29~items with $\geq 3$ broken variants and
11~items with 1--2 broken variants. The remaining
\textbf{59 verified-clean items} have variant answers confirmed by
model consensus and form the basis of all reported results. We report
the full 100-item results (before verification) in
Appendix~\ref{app:unverified} for transparency.

\subsection{Models}
We evaluate five models spanning three organizations:

\begin{itemize}[nosep, leftmargin=*]
\item \textbf{Llama~3.1 8B} (Meta, dense, 8B parameters)
\item \textbf{Llama~4 Scout 17B} (Meta, MoE, 17B active parameters)
\item \textbf{Qwen3 32B} (Alibaba, dense, 32B parameters)
\item \textbf{GPT-OSS 120B} (OpenAI, dense, 120B parameters)
\item \textbf{Llama~3.3 70B} (Meta, dense, 70B parameters)
\end{itemize}

\noindent GPT-OSS 120B serves as a \emph{negative control}: its training
data is known to exclude GSM8K, so any observed gap should reflect
difficulty effects rather than contamination. All models are accessed
via the Groq inference API at temperature $T = 0.0$ (greedy decoding)
with chain-of-thought prompting and \verb|\boxed{}| answer extraction,
with fallback to \verb|####|-delimited and last-number patterns.
Extraction rates exceed 94\% for all models on both original and
variant items, confirming that any accuracy gap is not an artifact
of differential extraction. Qwen3, GPT-OSS, and Llama~70B were
partially evaluated due to API daily token limits; both Llama~8B and
Scout models completed full evaluation on all items.

\subsection{Protocol}
Each model receives every original item and every variant item exactly
once ($T = 1$ trial per item, greedy decoding). We compute the
item-level contamination delta $\delta_i$ as the difference between
the model's accuracy on the original item and its mean accuracy across
the item's $\tau$-isomorphic variants. The model-level metric
$\Delta_{\mathrm{raw}}$ is the mean of item-level deltas.
We report clustered bootstrap 95\% confidence intervals
(10{,}000 resamples at the original-item level), analytic standard
errors $\mathrm{SE}(\Delta_{\mathrm{raw}}) = \sigma(\delta_i) / \sqrt{N}$,
and $p$-values from paired $t$-tests (testing
$H_1{:}\ \Delta_{\mathrm{raw}} > 0$).

\paragraph{Four of five models show no significant contamination.}
Llama~4 Scout 17B, Qwen3 32B, GPT-OSS 120B, and Llama~3.3 70B all
exhibit small, statistically non-significant accuracy gaps between
original and variant items ($\Delta_{\mathrm{raw}} \in [+0.049, +0.066]$,
all $p > 0.05$). These gaps are consistent with the 5--15 percentage
point baseline difficulty increase from numeric perturbation reported
in GSM-Symbolic~\cite{mirzadeh2024gsmsymbolic}. The negative control
(GPT-OSS 120B, $\Delta_{\mathrm{raw}} = +0.053$, $p = 0.174$)
confirms that the $\tau$-isomorphic variants are not systematically
harder in a way that would create false contamination signals.

\paragraph{Llama~3.1 8B shows a mild gap.}
The oldest model in our evaluation exhibits a statistically significant
but modest accuracy gap: 94.9\% on originals versus 82.3\% on variants
($\Delta_{\mathrm{raw}} = +0.126$, bootstrap 95\%
CI: $[+0.051, +0.202]$, $p = 0.002$). This 12.6-point gap exceeds
the baseline difficulty effect observed in the control model, suggesting
a mild memorization component. However, the effect is substantially
smaller than previously reported for similar model families (8--13\%
in GSM1K~\cite{zhang2024gsm1k}), and the model still achieves 82.3\%
accuracy on variants, indicating that the majority of its mathematical
reasoning transfers to novel numeric contexts.

\paragraph{Data quality as a confound.}
The dramatic difference between the unverified results
($\Delta_{\mathrm{raw}} \approx +0.42$ uniformly across all models)
and the verified-clean results ($\Delta_{\mathrm{raw}} \approx +0.06$)
underscores the importance of independent answer verification in
benchmark construction. Of the 40 items removed by verification,
29 had $\geq 3$ variants with provably wrong expected answers due
to annotation-chain replay bugs. Models that reasoned correctly on
these variants were penalized, creating an artificial contamination
signal that was uniform across all models --- including the control.
This finding has implications for other benchmark perturbation
approaches that rely on automated answer computation.

\paragraph{Comparison to prior work.}
Our verified-clean results are consistent with
GSM1K~\cite{zhang2024gsm1k}, which reported 8--13\% performance
drops for open-source models. The $\tau$-isomorphic approach provides
a complementary methodology with stronger structural guarantees ---
variants differ only in numeric values, not in problem framing ---
but requires careful answer verification to avoid data quality
confounds.

\subsection{Limitations}
\label{sec:limitations}
Several caveats apply to these results. First, the verification
procedure reduced the evaluation set from 100 to 59~items, reducing
statistical power; re-evaluation on a fully verified dataset of
$\geq 100$~items is needed to confirm the mild-gap finding for
Llama~3.1 8B and to detect smaller effects. Second, the evaluation
uses $T = 1$ trial per item (greedy decoding), preventing estimation
of per-item response variance; future work should use $T \geq 5$ to
compute proper IRT item parameters. Third, three of five models
(Qwen3, GPT-OSS, Llama~70B) were only partially evaluated due to
API daily token limits, yielding small effective sample sizes ($N = 8$
to $38$). Fourth, the multi-model consensus verification relies on
the assumption that multiple models solving a problem correctly is
strong evidence that the expected answer is correct; while this
assumption held for all manually-verified cases in our audit, it is
not a formal proof of answer correctness. Fifth, the annotation-chain
replay bugs identified in this work suggest that the variant generation
pipeline requires additional safeguards before the full dataset can
be considered production-ready.
